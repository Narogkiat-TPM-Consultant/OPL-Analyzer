A. ตกผลึกเทคนิคการเขียน OPL ที่เข้าใจใน 5 นาที

สกัดจาก Make OPL Sheet (JIPM), Example of OPL Japan (Yokohama) และ IKEA Concept (SCG) — ใช้เป็นฐานความรู้ของเกณฑ์ประเมินและ AI Feedback

1 แผ่น = 1 ประเด็น = 5 นาที
5分間教育 (go-fun-kan kyōiku = การศึกษา 5 นาที). ถ้าเกิน → แตกเป็นหลายใบ
ภาพ 80% : ข้อความ 20%
ทดสอบ: ปิดข้อความแล้วยังเข้าใจ = ผ่าน
สี 70 : 25 : 5
พื้นหลัง 70% · เนื้อหา 25% · สีเน้น 5% ที่จุดสำคัญสุดจุดเดียว
ปรัชญาแกน: "เปลี่ยนตัวอย่างความล้มเหลวให้เป็นสมบัติ" (Change the failure example to the treasure) — 1 แผ่น/คน/เดือน, ประกาศหน้าบอร์ดเช้า 3 นาที/แผ่น

เทคนิคแยกตามประเภท (3 ประเภทตาม JIPM)

ประเภทเป้าหมายโครงเนื้อหาที่ต้องมีสไตล์ภาพที่เหมาะ
Basic Knowledge
基礎知識
Kiso-Chishiki
 สอนหลักการ/วิธีใช้/มาตรฐานพื้นฐาน ให้ทำถูกตั้งแต่แรก นิยาม/หลักการถูกต้อง → ขั้นตอน 1-2-3 → "ทำไมสำคัญ / ถ้าทำผิดเกิดอะไร" IKEA Line-art (ขั้นตอน) + Cross-section (หลักการกลไก)
Kaizen Case
改善事例
Kaizen-Jirei
 เผยแพร่การปรับปรุงให้คนอื่นทำตาม (Yokoten 横展開) Before (ปัญหา+Loss) → ไอเดีย/กลไกที่ปรับ → After + ผลเป็นตัวเลข Same-angle Before/After + Cross-section กลไกที่เปลี่ยน
Trouble Case
トラブル事例
Trouble-Jirei
 กันการเกิดซ้ำของปัญหา/เครื่องเสีย อาการ (ภาพจุดจริง) → Why-Why ถึงราก → มาตรการแก้ + กันซ้ำ (Poka-yoke/มาตรฐาน) Photo+Annotation (จุดจริงวงแดง) + Visual Why-Why
กฎทอง 7 ข้อ: (1) 1 ประเด็น/แผ่น (2) ภาพ 80:20 (3) สีเน้นจุดเดียว (4) ทุกภาพมีลูกศร/วงกลมชี้จุด (5) มีคู่เทียบ OK/NG หรือ Before/After (6) อักษรใหญ่-เข้ม-คำง่าย (7) มีตาราง tracking + ขยายผล Yokoten

B.2 Component & State Spec

Component (Bootstrap 5)หน้าที่
Sidebar (offcanvas-lg)รายการ OPL ที่เคยตรวจ + คะแนน badge + ปุ่ม "ตรวจใหม่"
Upload dropzone (card + JS drag)ลากไฟล์ OPL (รูป/PDF) เริ่ม Agent 1
Chat thread (.bub)ฟอง AI / User / System; ปุ่ม inline action
Score chip + progress barคะแนนรวม + แยกราย Section จาก Rubric
Model selector (list-group)เลือก 1 ใน 4 model ก่อนถาม
Toast / spinnerสถานะ "กำลังวิเคราะห์…"
Stateสิ่งที่แสดง
1. Emptydropzone กลางจอ + ข้อความชวนอัปโหลด
2. Uploadingthumbnail + progress + ยกเลิกได้
3. Analyzingspinner "Agent 1→2 กำลังตรวจตาม Rubric…"
4. Feedbackscore chip + bubble เรียงจุดต้องปรับ (สำคัญสุดก่อน)
5. Refine (chat)ผู้ใช้ถามต่อ → Agent 3 ตอบอ้าง Best Practice
6. Acceptedปุ่ม "รับคำแนะนำ" → บันทึกเวอร์ชัน + ส่งเข้า workflow approval เดิม
หลัก UX: เรียง feedback จาก "คะแนนหายมากสุด" ก่อนเสมอ · ทุกคำแนะนำต้องมีตัวอย่าง "เขียนใหม่ให้กระชับ" · ปุ่มสร้าง Prompt วาดรูปแทรกเมื่อจุดอ่อนคือเรื่องภาพ (S3) · ธีมเขียวเดียวกับระบบเดิม

B.4 ตัวอย่างบทสนทนา Feedback (3 ประเภท)

Basic Knowledge — "วิธีอ่านเกจวัดแรงดัน"
AI: คะแนน 68/100 🟡 — โฟกัส 1 ประเด็นดีแล้ว แต่ S3 ภาพ = 12/25
• ภาพเกจเป็นรูปถ่ายไกล ไม่เห็นเข็ม → เพิ่ม Close-up + วงแดงช่วงปลอดภัย (เขียว) / อันตราย (แดง)
• ขั้นตอนเป็นพรืด → ทำเป็น 1-2-3 แบบ IKEA: "①มองตรงเข็ม ②อ่านช่วงเขียว ③เกินแดง=แจ้งหัวหน้า"
Kaizen Case — "ไคเซ็นชุดจับชิ้นงานลดเวลา setup"
AI: คะแนน 72/100 🟡 — ไอเดียดี แต่ S2.3 ผลตัวเลข = 2/6
• เขียน "เร็วขึ้นเยอะ" → "Setup 12 นาที → 4 นาที (-67%), 20 ครั้ง/วัน = เซฟ 2.7 ชม./วัน"
• Before/After คนละมุม → ถ่ายมุมเดียวกัน วางคู่กัน เน้นจุดที่เปลี่ยนด้วยวงแดง (Same-angle)
Trouble Case — "มอเตอร์ปั๊มไหม้ Line 3"
AI: คะแนน 54/100 🔴 — S2.2 Why-Why = 3/8 หยุดที่ "มอเตอร์ร้อน"
• ถาม Why ต่อ: ร้อน→ระบายความร้อนตัน→ฝุ่นเกาะครีบ→ไม่มีรอบทำความสะอาด → ราก: ขาดมาตรฐาน CIL
• มาตรการเป็นแค่ "เปลี่ยนมอเตอร์" → เพิ่มกันซ้ำ: รอบเป่าฝุ่นครีบทุกสัปดาห์ + เพิ่มใน AM Check Sheet

C. คลัง Prompt วาดรูป GPT Image 2.0

เมื่อจุดอ่อนคือ "ภาพ" (S3) Agent 3 จะแปลงเนื้อหา OPL เป็น Prompt สั่ง GPT Image 2.0 วาดภาพประกอบที่เข้าใจง่าย — คัดลอกไปใช้ได้ทันที ([…] = ช่องเติมข้อมูล)

C.1 โครงสร้าง Prompt มาตรฐาน (6 ส่วน)

[STYLE] technical line illustration, flat 2D, clean white background, no clutter [SUBJECT] [สิ่งที่วาด เช่น mechanical seal ของปั๊มหอยโข่ง] [VIEW] [มุมมอง: cross-section / step-by-step / hand performing task] [ANNOTATION] numbered callouts 1-2-3, arrows pointing to [จุดสำคัญ], label parts in Thai [COLOR] grey = structure, blue = normal flow, RED = the ONE critical point (70:25:5 rule) [CONSTRAINT] minimal text, large readable labels, operator can understand in 5 seconds, IKEA-manual simplicity
เคล็ด: ระบุ "label in Thai" และ "no realistic shading" เพื่อให้ได้เส้นสะอาดแบบคู่มือ ไม่ใช่ภาพ render เหมือนจริงที่อ่านยาก

C.2 Template พร้อมใช้ 3 สไตล์ (ตามประเภท OPL)

Basic สไตล์ IKEA Line-art (ขั้นตอนการทำงาน)
Flat 2D line illustration, IKEA assembly-manual style, clean white background. Show a worker's hands performing: [งาน เช่น การใช้ไขควงวัดไฟเช็คไฟรั่ว]. Break into [3] numbered panels (①②③), one action per panel, big numbers. Thin black outlines only, no shading. Minimal Thai labels. Lots of white space. One small RED arrow on the key safety point. Simple, calm, easy to follow in 5 seconds.
Trouble / Basic สไตล์ Cross-Section กลไก
Engineering cross-section diagram, flat 2D cutaway, technical drawing style, white background. Subject: [ชิ้นส่วน เช่น mechanical seal + เพลาปั๊ม] shown in half-section with hatching on metal. Blue arrows = [การไหลปกติ เช่น น้ำหล่อเย็น]. RED zone + red arrow = [จุดเสีย เช่น รอยรั่ว]. Numbered callouts with thin leader lines, part names in Thai. Grey structure, blue normal, red = single failure point. No photoreal rendering. Labels large and legible for shop-floor.
Kaizen สไตล์ Before / After มุมเดียวกัน
Two side-by-side flat 2D illustrations, SAME viewing angle, white background, labeled "ก่อน (Before)" and "หลัง (After)". Scene: [จุดงาน เช่น ชุดจับชิ้นงานบนแท่นเครื่อง]. Before = [สภาพเดิม/ปัญหา]; After = [หลังไคเซ็น]. Circle the CHANGED part with a RED ring in the After panel + short Thai callout of the gain. Clean lines, minimal text, no shading. Make the improvement obvious at a glance.
Quality check ก่อนใช้ภาพ: ① สีเน้นแดงมีจุดเดียว? ② มีลูกศร/เลขชี้จุด? ③ ปิด label แล้วยังเข้าใจ? ④ อักษรใหญ่อ่านจากที่ยืน? — ผ่านครบ = ใส่ใน OPL ได้
